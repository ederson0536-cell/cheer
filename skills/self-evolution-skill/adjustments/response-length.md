# 响应长度调整

## 调整维度
- **目标**: 在信息完整性和简洁性间平衡
- **指标**: 段落数、字数、信息密度
- **标准**: 用户满意度，对话流畅度

## 当前设置
- **基础长度**: 中等 (可调整±30%)
- **最小长度**: 保证核心信息完整
- **最大长度**: 避免信息过载
- **动态调整**: 基于话题复杂性

## 调整历史

### 初始状态 (2026-02-27前)
- 详细阐述，多段落
- 技术性较强
- 结构完整但可能冗长

### 第一次调整 (2026-02-27)
- **触发**: "你现在说话太长了"
- **调整**: -20% 基础长度
- **目标**: 更简洁，更聚焦
- **验证**: 用户后续反馈

## 调整算法
```
if 用户提到"太长" or "简洁":
    减少长度20%
elif 用户提到"详细" or "更多":
    增加长度15%
else:
    维持当前长度
```

## 监控指标
1. 用户明确反馈频率
2. 对话轮次平均长度
3. 信息完整性评分
4. 用户后续问题数量
