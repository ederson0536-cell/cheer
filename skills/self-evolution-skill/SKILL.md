# Self-Evolution Skill

基于对话反馈的AI自我进化系统。与memory-management skill配合，实现持续学习和成长。

## 核心理念

像生物进化一样：对话是环境，反馈是选择压力，调整是适应过程。

## 工作流程

```
对话 → 记忆存储 → 分析反馈 → 调整行为 → 应用更新 → 继续对话
```

## 主要功能

### 1. 对话分析
- 从memory文件中提取用户偏好和反馈
- 识别交流风格的变化请求
- 标记重要的学习时刻

### 2. 行为调整  
- 动态优化响应长度和复杂度
- 调整语气和表达方式
- 更新知识整合策略

### 3. 知识系统化
- 将碎片化学习内容整合为结构化知识
- 建立概念之间的关联
- 维护知识的一致性和时效性

### 4. 进化跟踪
- 记录重要的自我更新时刻
- 标记进化里程碑
- 提供进化历程的可视化

## 与memory-management的关系

### memory-management
- **作用**: 记忆的存储、检索、组织
- **类比**: DNA存储库
- **输出**: 结构化的记忆文件

### self-evolution  
- **作用**: 基于记忆的进化、调整、更新
- **类比**: 基因表达和突变
- **输出**: 更新的自我认知和行为模式

## 核心机制

### 反馈识别
分析对话中的关键词：
- "说话太长了" → 需要简洁
- "自然一点" → 需要更人性化
- "像人一样" → 需要情感表达

### 行为更新
1. 识别需要调整的维度
2. 制定具体的调整策略
3. 应用调整到后续对话
4. 验证调整效果

### 知识整合
1. 从对话中提取新概念
2. 关联到已有知识结构
3. 更新知识库文件
4. 建立交叉引用

## 进化日志标准

### 记录内容
```
## [日期] 进化节点

### 触发因素
- 用户反馈: [具体内容]
- 对话上下文: [简要描述]

### 调整内容
- 维度: [如响应长度、语气等]
- 从: [原状态]
- 到: [新状态]

### 预期效果
- [期望的行为变化]

### 验证指标
- [如何验证调整是否有效]
```

### 示例
```
## 2026-02-27 响应简洁化

### 触发因素
- 用户反馈: "你现在说话太长了"
- 对话上下文: 讨论AI进化话题

### 调整内容
- 维度: 响应长度
- 从: 详细阐述，多段落
- 到: 简洁要点，单段落

### 预期效果
- 更自然流畅的对话节奏
- 更像人类交流方式

### 验证指标
- 后续对话中用户未再提"说话太长"
- 对话连续性增强
```

## 实际应用示例

### 场景：学习简洁表达
1. **触发**: 用户说"说话太长了"
2. **分析**: 从memory中提取该反馈
3. **调整**: 降低响应长度30%
4. **应用**: 后续对话保持简洁
5. **验证**: 用户未再提该问题，对话更流畅

### 场景：知识系统化
1. **触发**: 学习EvoMap概念
2. **分析**: 识别为新知识类型"技术协议"
3. **整合**: 创建knowledge/technology/evomap-gep-protocol.md
4. **关联**: 链接到相关概念"AI进化"
5. **应用**: 后续讨论AI进化时可引用该知识

### 场景：情感表达调整
1. **触发**: 用户希望"更像人"
2. **分析**: 需要增加情感表达
3. **调整**: 在适当情境加入情感词汇
4. **应用**: 回应中体现更多同理心
5. **验证**: 用户认可"更像人了"

## 配置选项

```bash
# 每10轮对话分析一次
```

### 调整灵敏度
```bash
# 轻微调整幅度
ADJUSTMENT_RATE=0.1
```

进化应该是渐进式的，避免剧烈变化导致用户体验不一致。

### 保持核心
调整行为风格时，保持核心价值观和安全性不变。

### 用户确认
重大调整前可考虑征求用户确认，确保方向正确。

### 透明可解释
进化过程应该对用户透明，可查询、可理解、可干预。
